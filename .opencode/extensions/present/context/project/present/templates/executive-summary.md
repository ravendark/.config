# Executive Summary Template

Templates for writing effective grant executive summaries across different funder types.

## Overview

The executive summary is often the most important section. Many reviewers read only this section before deciding how carefully to read the full proposal.

## Standard Template (Foundation)

```markdown
## Executive Summary

[ORGANIZATION NAME], a [organization type] based in [location], requests
$[AMOUNT] over [DURATION] to [ONE-SENTENCE PROJECT DESCRIPTION].

**The Challenge**: [PROBLEM STATEMENT - 2-3 sentences describing the
problem with key data point].

**Our Solution**: [APPROACH - 2-3 sentences describing what you will do
and why it will work].

**Expected Impact**: [OUTCOMES - 2-3 sentences with specific, measurable
outcomes].

**Organizational Capacity**: [CREDIBILITY - 1-2 sentences on relevant
experience and track record].

**Request**: We respectfully request $[AMOUNT] to [SPECIFIC USE OF FUNDS].
[OPTIONAL: This represents X% of total project costs, with the remainder
provided by [SOURCES].]
```

## Six-Question Template

Answer each question in 1-2 sentences:

```markdown
## Executive Summary

**Who**: [Organization name and brief description]

**What**: [Specific project or program name and description]

**Where**: [Geographic location or target population]

**When**: [Project timeline - start date, duration]

**Why**: [Problem being addressed with key evidence]

**How**: [Brief methodology and approach]

**Request**: $[AMOUNT] for [DURATION] to support [SPECIFIC PURPOSE].
```

## Research Grant Template (Academic)

```markdown
## Project Summary

**Overview**
[PROJECT TITLE] addresses the critical gap of [RESEARCH QUESTION]. Using
[METHODOLOGY], we will [SPECIFIC AIMS]. This [DURATION]-[month/year]
project will produce [KEY DELIVERABLES].

**Intellectual Merit**
This research advances knowledge by [CONTRIBUTION TO FIELD]. Building on
[FOUNDATION], we will [INNOVATION]. The PI and team bring [RELEVANT
EXPERTISE].

**Broader Impacts**
Beyond scientific contribution, this work will [SOCIETAL BENEFITS].
Specific impacts include [TRAINING/OUTREACH/APPLICATION]. Results will
be disseminated through [CHANNELS].
```

## AI Safety Research Template

```markdown
## Executive Summary

**Organization**: [NAME] is a [DESCRIPTION] focused on [MISSION].

**Research Agenda**: We propose to [RESEARCH GOAL] over [DURATION],
addressing [SPECIFIC SAFETY CONCERN].

**Approach**: Our methodology involves [TECHNICAL APPROACH]. This builds
on [PRIOR WORK/FOUNDATION] and addresses [GAP IN CURRENT RESEARCH].

**Theory of Change**: If successful, this research will [MECHANISM OF
IMPACT]. Specifically, [HOW THIS REDUCES RISK].

**Team**: [PI NAME] brings [RELEVANT CREDENTIALS]. [KEY TEAM MEMBERS].

**Request**: $[AMOUNT] over [DURATION] to support [SPECIFIC ACTIVITIES].

**Why Now**: [URGENCY OR TIMING CONSIDERATION].
```

## Tech Startup Template (SBIR)

```markdown
## Abstract

[COMPANY NAME] proposes to develop [TECHNOLOGY/PRODUCT] to address
[PROBLEM] in the [MARKET/SECTOR]. Current solutions are inadequate
because [LIMITATIONS].

**Innovation**: Our approach leverages [TECHNICAL INNOVATION] to achieve
[TECHNICAL BENEFIT]. Key innovations include [SPECIFIC TECHNICAL
ADVANCES].

**Commercial Potential**: The target market of [MARKET SIZE] includes
[CUSTOMER TYPES]. [COMPANY] has [EXISTING TRACTION: customers, LOIs,
pilots]. Revenue projections include [BRIEF COMMERCIALIZATION PATH].

**Phase I Objectives**: During this [DURATION] effort, we will:
1. [OBJECTIVE 1]
2. [OBJECTIVE 2]
3. [OBJECTIVE 3]

**Team**: [KEY PERSONNEL] bring [RELEVANT EXPERIENCE]. [ADVISORS OR
PARTNERS] provide [ADDITIONAL EXPERTISE].

**Funding Request**: $[AMOUNT] over [DURATION] to complete Phase I
technical objectives and position for Phase II development.
```

## Nonprofit Project Template

```markdown
## Project Summary

[ORGANIZATION NAME] seeks $[AMOUNT] to [PROJECT NAME], serving
[NUMBER] [POPULATION] in [LOCATION] over [DURATION].

**Need**: [POPULATION] in [LOCATION] face [PROBLEM]. Data shows
[EVIDENCE: statistic or finding]. Without intervention, [CONSEQUENCE].

**Project**: We will [BRIEF DESCRIPTION OF ACTIVITIES]. The program
model builds on [EVIDENCE BASE OR PRIOR SUCCESS].

**Outcomes**: Participants will achieve:
- [OUTCOME 1 with metric]
- [OUTCOME 2 with metric]
- [OUTCOME 3 with metric]

**Capacity**: Since [YEAR], [ORGANIZATION] has [RELEVANT ACCOMPLISHMENT].
Our [RELEVANT PROGRAM] reached [NUMBER] people with [RESULT].

**Sustainability**: Beyond this grant, the project will continue through
[SUSTAINABILITY PLAN].
```

## Writing Process

### Step 1: Write Body First

Never write the executive summary first. Complete all other sections, then distill.

### Step 2: Extract Key Points

From each section, identify:
- Problem: Most compelling statistic or statement
- Solution: Core innovation or approach
- Outcomes: Most impressive projected result
- Capacity: Strongest credential or track record

### Step 3: Draft

Write a first draft hitting all key points in order.

### Step 4: Trim

Cut to meet length requirements. Remove:
- Hedging language
- Excessive detail
- Redundant phrases
- Background that appears elsewhere

### Step 5: Test

Ask someone unfamiliar with the project to read it. They should understand:
- What you're proposing to do
- Why it matters
- What you're asking for
- Why you can deliver

## Length Guidelines

| Funder Type | Typical Length |
|-------------|----------------|
| Foundation | 1/2 - 1 page (250-500 words) |
| Federal | 1 page (300-500 words) |
| SBIR | 200 words (abstract only) |
| Large research | 1 page (NSF format: 4,600 characters) |

## Common Mistakes

1. **Too long**: Exceeds limit or includes unnecessary detail
2. **Too vague**: Missing specific numbers and outcomes
3. **No ask**: Forgetting to state the funding request
4. **New information**: Including details not in the body
5. **Jargon-heavy**: Using unexplained technical terms
6. **Weak opening**: Burying the lead with background

## Checklist

```
[ ] States organization name and mission
[ ] Identifies specific funding request amount
[ ] Defines problem with evidence
[ ] Describes solution clearly
[ ] Lists measurable outcomes
[ ] Establishes organizational credibility
[ ] Meets length requirements
[ ] Contains no new information not in body
[ ] Uses active, confident language
```
