# Submission Checklist Template

Pre-submission verification checklists for grant proposals.

## Overview

Use these checklists to ensure proposals are complete before submission. Customize based on specific funder requirements.

## Universal Checklist

### Before Writing

```
[ ] Confirmed eligibility (organization type, location, etc.)
[ ] Noted deadline (submission vs. receipt deadline?)
[ ] Downloaded current application forms
[ ] Reviewed evaluation criteria
[ ] Identified page/character limits
[ ] Checked formatting requirements
[ ] Confirmed budget limits and restrictions
[ ] Identified required attachments
[ ] Set internal deadlines (draft, review, final)
```

### Content Completeness

```
[ ] Executive summary / abstract
[ ] Organization background / qualifications
[ ] Statement of need with evidence
[ ] Project description / narrative
[ ] Goals and objectives (SMART format)
[ ] Timeline / milestones
[ ] Evaluation plan
[ ] Budget (matches narrative)
[ ] Budget justification
[ ] Sustainability plan (if required)
[ ] Key personnel / biographical information
```

### Formatting Verification

```
[ ] Correct font and size
[ ] Correct margins
[ ] Within page limits
[ ] Within character/word limits
[ ] Page numbers present
[ ] Headers/footers per requirements
[ ] Consistent formatting throughout
[ ] Required sections in correct order
```

### Required Attachments

```
[ ] Letters of support / commitment
[ ] Resumes / CVs / biosketches
[ ] Organizational documents (as required):
    [ ] 501(c)(3) determination letter
    [ ] Board list
    [ ] Audited financials
    [ ] Annual report
[ ] Data management plan (if required)
[ ] IRB approval or pending status
[ ] Other: [funder-specific]
```

### Technical Requirements

```
[ ] Correct file format (PDF, Word, etc.)
[ ] File naming convention followed
[ ] File size within limits
[ ] All forms completed and signed
[ ] Registration current (SAM, Grants.gov, etc.)
[ ] Authorized signatory identified
```

### Final Review

```
[ ] Proofread by someone who has not seen it
[ ] Math verified (budget totals, statistics)
[ ] Citations complete and accurate
[ ] Acronyms defined on first use
[ ] Active voice used throughout
[ ] Specific numbers (not "many" or "significant")
[ ] All questions in RFP addressed
[ ] Proposal reads well from funder perspective
```

## Federal Grants Checklist

### NSF Specific

```
PRE-SUBMISSION:
[ ] NSF ID current
[ ] Verified program solicitation is current
[ ] COA (Collaborators and Other Affiliations) updated
[ ] Single Copy Documents prepared

REQUIRED SECTIONS:
[ ] Cover Sheet (completed in Research.gov)
[ ] Project Summary (1 page, 3 sections)
    [ ] Overview
    [ ] Intellectual Merit
    [ ] Broader Impacts
[ ] Project Description (15 pages max)
[ ] References Cited
[ ] Biographical Sketches (3 pages each)
[ ] Budget and Budget Justification
[ ] Current and Pending Support
[ ] Facilities, Equipment, and Other Resources
[ ] Data Management Plan (2 pages)

OPTIONAL:
[ ] Supplementary Documents (if allowed)
[ ] Letters of Collaboration (not support)
[ ] Postdoctoral Mentoring Plan (if applicable)
```

### NIH Specific

```
PRE-SUBMISSION:
[ ] eRA Commons ID active for all key personnel
[ ] Institutional profile current
[ ] Verified FOA is current

REQUIRED (R01):
[ ] SF424 forms complete
[ ] Project Summary/Abstract (30 lines)
[ ] Specific Aims (1 page)
[ ] Research Strategy (12 pages)
    [ ] Significance
    [ ] Innovation
    [ ] Approach
[ ] Bibliography and References Cited
[ ] Budget (modular or detailed)
[ ] Biosketches (5 pages each)
[ ] Facilities and Other Resources
[ ] Equipment

ADDITIONAL AS APPLICABLE:
[ ] Human Subjects section
[ ] Vertebrate Animals section
[ ] Resource Sharing Plans
[ ] Authentication of Key Biological Resources
[ ] Letters of Support
[ ] Multiple PD/PI Leadership Plan
```

### SBIR/STTR Specific

```
ELIGIBILITY:
[ ] Small business size verified
[ ] US ownership and operation confirmed
[ ] PI employment requirement met
[ ] Research institution identified (STTR only)

REQUIRED:
[ ] Cover page with all certifications
[ ] Technical proposal within page limit
[ ] Abstract (200 words)
[ ] Commercialization plan
[ ] Budget and justification
[ ] Key personnel biosketches
[ ] Facilities and equipment description
[ ] Subcontracts/consultants documented
[ ] Prior SBIR/STTR awards disclosed
[ ] Company information
```

## Foundation Checklist

### Private Foundation

```
PRE-SUBMISSION:
[ ] Confirmed invitation to apply (if LOI required)
[ ] Deadline confirmed (often less strict)
[ ] Contact with program officer (if allowed)

COMMON REQUIREMENTS:
[ ] Cover letter
[ ] Executive summary (1/2-1 page)
[ ] Full proposal narrative (5-10 pages typical)
[ ] Budget (1-2 pages)
[ ] Budget narrative
[ ] Timeline
[ ] Evaluation plan

ATTACHMENTS (TYPICAL):
[ ] 501(c)(3) determination letter
[ ] List of board of directors
[ ] Organizational budget
[ ] Most recent audited financials
[ ] Most recent Form 990
[ ] Key staff resumes
[ ] Letters of support (limited)
```

### Community Foundation

```
[ ] Geographic eligibility confirmed
[ ] Specific fund requirements checked
[ ] Online portal account created
[ ] Character limits noted (portal fields)
[ ] Attachments uploaded correctly
[ ] Contact information current
```

## Letter of Inquiry (LOI) Checklist

```
[ ] Within page/word limit
[ ] Organization introduction (1-2 sentences)
[ ] Problem statement with evidence
[ ] Proposed solution (brief)
[ ] Requested amount and duration
[ ] Brief qualifications
[ ] Contact information
[ ] Follows funder's specific LOI format

COMMON MISTAKES TO AVOID:
[ ] Not too long (LOI should be brief)
[ ] Not asking for wrong amount
[ ] Not addressing wrong program area
[ ] Not including all requested info
```

## Partner/Subaward Checklist

```
FOR EACH PARTNER:
[ ] Letter of commitment (not just support)
[ ] Letter describes specific role
[ ] Letter references proposed project
[ ] Budget provided for their portion
[ ] Scope of work defined
[ ] Authorized signature on letter
[ ] Partner's required documents collected
```

## Last-Minute Verification

### 24 Hours Before

```
[ ] Final proofread complete
[ ] All signatures obtained
[ ] Budget totals verified
[ ] Page counts confirmed
[ ] File converted to required format
[ ] All attachments assembled
[ ] Submission portal tested
```

### Submission Day

```
[ ] Adequate time before deadline
[ ] Stable internet connection
[ ] Backup submission plan (if portal fails)
[ ] Confirmation receipt saved
[ ] Copy archived for records
```

### After Submission

```
[ ] Confirmation received and saved
[ ] Team notified of submission
[ ] Grant tracking system updated
[ ] Calendar marked for notification date
[ ] Thank you to collaborators sent
[ ] Lessons learned documented
```

## Common Rejection Reasons

```
AVOID THESE:
[ ] Submitted after deadline
[ ] Missing required sections
[ ] Over page/budget limits
[ ] Wrong formatting
[ ] Incomplete forms
[ ] Missing attachments
[ ] Ineligible organization
[ ] Project outside program scope
[ ] Unresponsive to RFP questions
[ ] Math errors in budget
```
