# Evaluation Plan Template

Templates for comprehensive program evaluation sections in grant proposals.

## Overview

An evaluation plan demonstrates how you will know if your project succeeds. Strong plans align with objectives, use appropriate methods, and include realistic timelines.

## Standard Evaluation Plan Template

```markdown
## Evaluation Plan

### Overview
[ORGANIZATION] will conduct a [formative/summative/mixed] evaluation to assess
progress toward objectives and measure project outcomes. The evaluation will be
led by [internal staff / external evaluator] with a budget of $[AMOUNT] ([X]%
of total project budget).

### Evaluation Questions
1. [Process question]: To what extent are activities implemented as planned?
2. [Outcome question]: Do participants achieve [specific outcome]?
3. [Impact question]: What is the longer-term effect on [population/system]?

### Evaluation Design
[Brief description of overall approach: pre-post, quasi-experimental, etc.]

### Data Collection Methods

| Question | Data Source | Method | Timing | Sample |
|----------|-------------|--------|--------|--------|
| Process | Staff | Activity logs | Ongoing | All activities |
| Outcome | Participants | Pre/post survey | Start/End | All participants |
| Outcome | Records | Data extraction | Quarterly | All participants |
| Impact | Stakeholders | Interviews | End | N=15 |

### Indicators and Targets

| Objective | Indicator | Baseline | Target | Source |
|-----------|-----------|----------|--------|--------|
| Objective 1 | [Metric] | [Value] | [Value] | [Source] |
| Objective 2 | [Metric] | [Value] | [Value] | [Source] |
| Objective 3 | [Metric] | [Value] | [Value] | [Source] |

### Analysis Plan
[Description of how data will be analyzed: descriptive statistics, comparison
of pre/post scores, thematic analysis of qualitative data, etc.]

### Reporting
Evaluation findings will be reported to [stakeholders] through:
- Quarterly progress reports to funder
- Annual report to board of directors
- Final evaluation report

### Use of Findings
[How evaluation results will inform program improvement and future work]
```

## Participatory Evaluation Template

```markdown
## Evaluation Plan

### Participatory Approach
[ORGANIZATION] is committed to participatory evaluation that centers
[community/participant] voice. Evaluation activities will be designed
and implemented with input from [stakeholder group].

### Community Advisory Role
[N] community members will serve on the Evaluation Advisory Committee, which will:
- Review evaluation design and instruments
- Interpret findings from community perspective
- Advise on dissemination of results

### Data Collection Methods

**Quantitative**:
- [Method 1]: [Description]
- [Method 2]: [Description]

**Qualitative (Community-Centered)**:
- Photovoice: Participants will document [aspect] through photography
- Story circles: Groups will share experiences with [topic]
- Community mapping: [Description of participatory mapping]

### Data Sovereignty
Data collected from [community] will be:
- Stored [location/method]
- Owned by [entity]
- Accessible to community upon request
- Published only with community consent
```

## Data Collection Instruments

### Pre-Post Survey Template

```markdown
### Survey Instrument Overview

**Pre-Survey** (administered at enrollment):
- Demographics (10 items)
- Baseline knowledge assessment (15 items)
- Self-efficacy scale (10 items)
- Prior experience (5 items)

**Post-Survey** (administered at program completion):
- Knowledge assessment (15 items - same as pre)
- Self-efficacy scale (10 items - same as pre)
- Satisfaction (10 items)
- Open-ended feedback (3 items)

**Follow-up Survey** (administered 3 months post):
- Behavior change items (10 items)
- Sustained knowledge (10 items)
- Application of skills (5 items)

All survey instruments will be pilot tested with [N] individuals from the
target population prior to use.
```

## Best Practices

1. **Align with objectives**: Every objective should have corresponding evaluation
2. **Be realistic**: Do not promise more evaluation than resources allow
3. **Build in time**: Include evaluation in project timeline
4. **Budget adequately**: Allocate 5-15% for evaluation
5. **Use mixed methods**: Combine quantitative and qualitative when possible
6. **Plan for use**: Specify how findings will inform improvement
7. **Address ethics**: Include IRB/consent considerations where relevant
