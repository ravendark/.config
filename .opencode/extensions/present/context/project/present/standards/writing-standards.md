# Grant Writing Standards

Style guide and conventions for professional grant proposals.

## Overview

Grant writing requires a balance between persuasion and precision. These standards ensure proposals are clear, credible, and compelling.

## Voice and Tone

### Active Voice (Required)

```
CORRECT: "We will train 50 teachers in evidence-based practices."
WRONG: "Fifty teachers will be trained in evidence-based practices."

CORRECT: "The research team will analyze data monthly."
WRONG: "Data will be analyzed on a monthly basis."
```

### Confident (Not Arrogant)

```
CORRECT: "Our track record demonstrates capability..."
WRONG: "We are the best organization for this work..."

CORRECT: "This approach builds on proven methods..."
WRONG: "This revolutionary approach will transform..."
```

### Professional (Not Stiff)

```
CORRECT: "We will partner with local schools..."
WRONG: "The applicant organization shall engage in collaborative
        endeavors with educational institutions..."
```

## Sentence Structure

### Length Guidelines

| Context | Target Length |
|---------|---------------|
| Average sentence | 15-20 words |
| Maximum sentence | 30 words |
| Paragraph | 3-5 sentences |
| Executive summary paragraph | 2-3 sentences |

### Paragraph Structure

```
TOPIC SENTENCE: Main point of paragraph
SUPPORT 1: Evidence or detail
SUPPORT 2: Additional evidence
TRANSITION: Link to next paragraph (when needed)
```

## Precision Standards

### Quantification

| Vague | Precise |
|-------|---------|
| many participants | 347 participants |
| significant improvement | 23% improvement |
| regularly | weekly / every Tuesday |
| soon | by March 2025 |
| most | 85% (17 of 20) |
| substantial | $50,000 / 40 hours |

### Specificity Scale

```
POOR: "We will help communities."
FAIR: "We will assist 10 communities in the region."
GOOD: "We will provide training to 10 rural communities in
       northern Maine."
BEST: "We will provide 40 hours of training in water quality
       monitoring to community health workers in 10 rural
       communities (population <5,000) in Aroostook County,
       Maine."
```

## Terminology Consistency

### Within Document

Pick one term and use it consistently:

```
| Choose One | Not Both |
|------------|----------|
| participants | participants / attendees / trainees |
| program | program / project / initiative |
| outcomes | outcomes / results / impacts |
| community partners | community partners / collaborators / stakeholders |
```

### With Funder

Use the funder's terminology:

```
NSF: "Broader Impacts" (not "community benefits")
NIH: "Specific Aims" (not "objectives")
Foundations: Match their language for priorities
```

## Evidence Standards

### Citation Formats

```
IN-TEXT: "Research shows that [finding] (Author, Year)."
FULL: "Author, A. (Year). Title. Journal, Volume, Pages."

ACCEPTABLE: "According to [Source], [finding] (Year)."
```

### Evidence Quality Hierarchy

1. Meta-analyses and systematic reviews
2. Randomized controlled trials
3. Quasi-experimental studies
4. Cohort studies and case-control studies
5. Program evaluation data
6. Expert opinion and reports
7. Organizational data and testimonials

### Recency Standards

| Field | Data Currency |
|-------|---------------|
| Rapidly changing (tech, policy) | Within 2 years |
| Stable (demographics, health) | Within 5 years |
| Historical context | Any date with context |

## Formatting Conventions

### Numbers

| Rule | Example |
|------|---------|
| Spell out 1-9 | three participants |
| Use numerals 10+ | 25 participants |
| Always numeral with units | 5 hours, 3 months |
| Start sentence spelled out | Twenty-five participants... |
| Large numbers use comma | 1,500 not 1500 |

### Lists

```
USE BULLETS WHEN:
- Items are parallel in structure
- Order does not matter
- 3 or more items

USE NUMBERS WHEN:
1. Sequence matters
2. Items will be referenced later
3. Steps in a process
```

### Emphasis

```
Use BOLD for:
- Section headers
- Key terms on first use
- Critical information

Use ITALICS for:
- Publication titles
- Emphasis (sparingly)
- Technical terms

AVOID:
- Underlining
- ALL CAPS
- Excessive bold
```

## Common Errors

### Redundancy

```
WRONG: "past history" -> "history"
WRONG: "future plans" -> "plans"
WRONG: "collaborate together" -> "collaborate"
WRONG: "currently ongoing" -> "ongoing"
```

### Hedging (Minimize)

```
WEAK: "We hope to achieve..."
STRONG: "We will achieve..."

WEAK: "The program may result in..."
STRONG: "The program will result in..."

WEAK: "We believe this approach might..."
STRONG: "This approach will..."
```

### Filler Words (Remove)

```
Remove: "In order to" -> "To"
Remove: "It is important to note that" -> (Delete)
Remove: "The fact that" -> (Restructure)
Remove: "It should be noted that" -> (Delete)
```

## Section-Specific Standards

### Executive Summary

- Write LAST, after all sections complete
- Include specific ask amount
- No new information not in body
- 250 words or less typically

### Budget Narrative

- Connect every cost to activities
- Show calculations clearly
- Use funder-required categories
- Match narrative exactly

### Evaluation Section

- Align metrics with objectives
- Be realistic about measurement
- Include both process and outcome
- Budget adequate resources

## Quality Checklist

```
[ ] Active voice throughout
[ ] All claims have evidence
[ ] Consistent terminology
[ ] Numbers are specific
[ ] Sections flow logically
[ ] Budget matches narrative
[ ] Character counts met
[ ] Funder requirements followed
```
