# Character Limits Reference

Funder-specific character and page limits for common proposal sections.

## Overview

Exceeding limits causes automatic rejection for many funders. Always verify current limits in the specific RFP/RFA.

## NSF (National Science Foundation)

| Section | Limit |
|---------|-------|
| Project Summary | 1 page (4,600 characters) |
| Project Description | 15 pages |
| References Cited | No limit |
| Biographical Sketch | 3 pages per person |
| Budget Justification | 5 pages |
| Data Management Plan | 2 pages |
| Facilities & Equipment | No limit |

### Project Summary Structure (NSF)

```
OVERVIEW (1 paragraph)
[Brief description of project - approximately 1,500 characters]

INTELLECTUAL MERIT (1 paragraph)
[Significance and contribution - approximately 1,500 characters]

BROADER IMPACTS (1 paragraph)
[Societal benefits - approximately 1,500 characters]
```

## Page vs. Character Calculations

### Standard Assumptions

```
Single-spaced, 12pt Times:
- 1 page = approximately 3,000 characters (with spaces)
- 1 page = approximately 500 words
- 1 page = approximately 45 lines

Double-spaced, 12pt Times:
- 1 page = approximately 1,500 characters
- 1 page = approximately 250 words
- 1 page = approximately 23 lines
```

### Word Count Conversions

| Words | Characters (approx) | Pages (single-spaced) |
|-------|---------------------|----------------------|
| 100 | 600 | 0.2 |
| 250 | 1,500 | 0.5 |
| 500 | 3,000 | 1 |
| 1,000 | 6,000 | 2 |
| 2,500 | 15,000 | 5 |

## Formatting Requirements

### Federal Standard Format

```
Font: 11pt Arial or 12pt Times New Roman
Margins: 1 inch all sides
Line spacing: Single-spaced (unless specified)
Headers: Within page count
Page numbers: Required, outside margins
```

### What Counts Toward Limits

| Typically Included | Typically Excluded |
|-------------------|-------------------|
| Body text | Cover pages |
| Tables within text | Table of contents |
| Figures within text | Bibliography/References |
| Headers/footers | Budget pages |
| Section headings | Letters of support |

## Best Practices

1. **Check the specific RFP**: Limits change; verify current requirements
2. **Build in buffer**: Aim for 90% of limit to allow for editing
3. **Use character count tools**: Word processors may count differently
4. **Test PDF conversion**: Final PDF may differ from source document
5. **Include white space**: Cramming text hurts readability
6. **Verify font rendering**: Some fonts expand in PDF

## Character Counting Tools

### In Word Processors

```
Microsoft Word: Review > Word Count
Google Docs: Tools > Word Count (Ctrl+Shift+C)
LibreOffice: Tools > Word Count
```

### Command Line

```bash
# Character count (with spaces)
wc -c file.txt

# Word count
wc -w file.txt

# Line count
wc -l file.txt
```
