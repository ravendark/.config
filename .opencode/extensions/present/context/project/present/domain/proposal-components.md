# Proposal Components

Standard sections found in most grant proposals, with guidance on purpose and content for each.

## Overview

| Component | Purpose | Typical Length |
|-----------|---------|----------------|
| Executive Summary | Quick overview for busy reviewers | 1/2-1 page |
| Organization Background | Establish credibility and capacity | 1-2 pages |
| Statement of Need | Define problem with evidence | 1-2 pages |
| Project Description | Detail approach and methodology | 3-10 pages |
| Budget | Itemize costs with justification | 1-3 pages |
| Timeline | Show project schedule | 1/2-1 page |
| Evaluation Plan | Define success metrics | 1-2 pages |
| Sustainability | Post-grant continuation | 1/2-1 page |
| Key Personnel | Team qualifications | 1-2 pages |

## Executive Summary

### Purpose
Provide a standalone overview that captures the entire proposal in 1-2 paragraphs. Many reviewers read this first (and sometimes only this).

### Structure
1. **Who**: Organization name and brief description
2. **What**: Specific project or program
3. **Where**: Geographic scope or target population
4. **When**: Project duration
5. **Why**: Problem being addressed
6. **How**: Brief methodology
7. **How much**: Funding request

### Best Practices
- Write this LAST, after completing other sections
- Keep under 250 words for most funders
- Make every word count
- Include the specific ask

## Organization Background

### Purpose
Establish credibility to execute the proposed project.

### Key Elements
- Founding date and mission
- Relevant past projects and outcomes
- Staff qualifications
- Geographic reach
- Financial stability indicators

### Best Practices
- Focus on relevant experience
- Include specific numbers and outcomes
- Avoid lengthy history unless relevant
- Highlight unique organizational capacity

## Statement of Need

### Purpose
Define the problem clearly and compellingly with evidence.

### Structure
1. **Problem definition**: What issue are you addressing?
2. **Evidence**: Data, research, testimonials supporting severity
3. **Root causes**: Why does this problem exist?
4. **Target population**: Who is affected?
5. **Gap**: What is not being addressed by current solutions?

### Best Practices
- Use recent, credible data
- Include local/specific data when available
- Avoid deficit-based framing of communities
- Connect problem to funder priorities
- Keep emotional appeals grounded in evidence

## Project Description

### Purpose
Detail your approach, methodology, and implementation plan.

### Key Elements
- **Goals**: Broad outcomes you hope to achieve
- **Objectives**: Specific, measurable targets (SMART format)
- **Methods/Activities**: What you will actually do
- **Logic Model**: If-then chain from activities to outcomes

### SMART Objectives
- **Specific**: Clear and well-defined
- **Measurable**: Quantifiable indicators
- **Achievable**: Realistic given resources
- **Relevant**: Connected to goals
- **Time-bound**: Clear deadlines

### Best Practices
- Use active voice
- Be specific about methods
- Distinguish goals, objectives, and activities
- Show logical connection between activities and outcomes

## Budget

### Purpose
Itemize all project costs with clear justification.

### Standard Categories
1. **Personnel**: Salaries, wages, fringe benefits
2. **Equipment**: Items over $5K (typically)
3. **Travel**: Project-related travel
4. **Supplies**: Consumables, materials
5. **Contractual**: Consultants, subcontracts
6. **Other Direct Costs**: Miscellaneous
7. **Indirect Costs**: Overhead (if allowed)

### Best Practices
- Every line item needs justification
- Show calculation methodology
- Ensure totals match narrative
- Follow funder budget format exactly
- Include cost share if required

## Timeline

### Purpose
Show project schedule and major milestones.

### Formats
- **Gantt chart**: Visual bar chart format
- **Milestone table**: Key dates and deliverables
- **Quarterly breakdown**: Activities by quarter

### Best Practices
- Be realistic about timing
- Account for delays and approval processes
- Show dependencies between activities
- Include reporting periods

## Evaluation Plan

### Purpose
Define how you will measure success and learn from the project.

### Key Elements
- **Process evaluation**: Are activities happening as planned?
- **Outcome evaluation**: Are objectives being met?
- **Impact evaluation**: Long-term change (if applicable)
- **Data collection methods**: Surveys, interviews, metrics
- **Reporting timeline**: When will data be collected and analyzed

### Best Practices
- Align metrics with objectives
- Include both quantitative and qualitative measures
- Plan for baseline data collection
- Budget for evaluation activities

## Sustainability

### Purpose
Show how the project will continue after grant funding ends.

### Strategies
- Revenue diversification
- Institutionalization within partner organizations
- Fee-for-service models
- Additional grant funding
- Policy or systems change

### Best Practices
- Be realistic about sustainability challenges
- Show concrete plans, not vague intentions
- Connect to organizational capacity
- Address funder concerns about dependency

## Key Personnel

### Purpose
Demonstrate team qualifications to execute the project.

### Key Elements
- Project director/PI qualifications
- Key staff roles and qualifications
- Advisory board (if applicable)
- Consultants and their roles
- Biographical sketches

### Best Practices
- Focus on relevant experience
- Include specific accomplishments
- Show complementary skill sets
- Indicate time commitment to project
