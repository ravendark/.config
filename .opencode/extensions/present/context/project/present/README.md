# Grant Writing Domain

This context provides support for research grant proposal development, budget planning, and funder-specific guidance.

## Overview

The grant extension enables structured grant proposal development with:

- **Funder Research**: Analyze funder priorities, eligibility requirements, and application formats
- **Proposal Drafting**: Template-based narrative section development
- **Budget Development**: Line-item budgets with justification patterns
- **Review and Submission**: Checklist validation and compliance verification

## Grant Types Supported

| Grant Type | Primary Funders | Focus |
|------------|-----------------|-------|
| AI Safety Research | Open Phil, SFF, LTFF | Technical clarity, existential risk framing |
| Tech Startup | SBIR, state programs | Commercialization path, market analysis |
| Nonprofit Project | Foundations, government | Mission alignment, community impact |
| Academic Research | NSF, NIH, private | Peer review, methodology rigor |

## Core Proposal Components

Every grant proposal includes these essential sections:

| Component | Description |
|-----------|-------------|
| Executive Summary | Concise overview (who, what, where, when, why, how) |
| Organization Background | History, mission, capacity |
| Statement of Need | Problem definition with data |
| Project Description | Goals, objectives, methodology |
| Budget | Line-item breakdown with justification |
| Timeline | Visual or tabular project schedule |
| Evaluation Plan | Success metrics and measurement |
| Sustainability | Post-grant continuation plan |
| Key Personnel | Team qualifications |

## Workflow Stages

### Stage 1: Funder Research
- Identify funder mission and priorities
- Verify eligibility requirements
- Note deadline and format requirements
- Gather past awarded grant examples

### Stage 2: Proposal Development
- Develop narrative sections using templates
- Track character limits and word counts
- Align with funder-specific terminology

### Stage 3: Budget Planning
- Create line-item budget
- Write budget justification for each item
- Verify totals and funding period alignment

### Stage 4: Review and Submission
- Complete pre-submission checklist
- Verify all links are functional
- Review required acknowledgements
- Validate character counts

## Best Practices

### Technical Clarity
- Describe technology in straightforward terms
- Avoid unnecessary jargon
- If reviewers do not understand in two passes, they move on

### Impact Statements
- Use 1-3 sentence format
- Focus on measurable outcomes
- Connect to funder mission

### Budget Justification
- Every line item needs rationale
- Connect expenses to project activities
- Explain calculation methodology
