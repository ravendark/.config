## Overview

Team-mode skills (skill-team-research, skill-team-plan, skill-team-implement) spawn teammates via TeammateTool without specifying `subagent_type`, making them eligible for `CLAUDE_CODE_FORK_SUBAGENT=1` prompt cache sharing. This plan restructures teammate prompts to maximize the byte-identical prefix across teammates, replaces the hardcoded `token_usage_multiplier: 5.0` with a dynamic formula reflecting cache savings, and updates supporting documentation. Both environment variables (`CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` and `CLAUDE_CODE_FORK_SUBAGENT=1`) are already enabled in global settings -- no env var changes are needed.

### Research Integration

Key findings from the research report (01_team-mode-fork-cache.md):

1. **Cache eligibility confirmed**: TeammateTool dispatches omit `subagent_type`, so FORK_SUBAGENT cache sharing applies. Teammates 2-N inherit the parent's cached prompt prefix at ~90% input token reduction.
2. **Prompt ordering is the primary lever**: Current teammate prompts intersperse shared content (task description, model preference, artifact number) with unique content (role, focus, output path). Reordering to put shared content first maximizes the cacheable prefix.
3. **Multiplier is wildly inaccurate**: The hardcoded 5.0x is based on pre-cache assumptions. With 70% shared prefix and 90% cache hit rate, a 4-teammate team costs ~2.4x (not 5.0x).
4. **Default team sizes are already appropriate**: Research uses 4 (hardcoded, optimal), plan uses 2, implement uses 2. No defaults need changing.
5. **skill-team-research hardcodes team_size=4**: This is intentional per the SKILL.md comment ("Team research always uses 4 teammates"). The `--team-size` flag is deliberately overridden, not bugged.
