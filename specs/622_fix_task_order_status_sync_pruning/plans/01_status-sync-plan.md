## Overview

Fix two confirmed defects in the agent system's Task Order lifecycle: (1) agents contaminating the Task Order section with artifact links because `artifact-linking-todo.md` lacks an explicit prohibition, and (2) extension implementer skills (lean, nvim) bypassing `update-task-status.sh` so Mode B terminal-transition regeneration never fires for those task types. The research report confirmed that Sub-issue 1 (Phase 3 grep patterns) needs no fix -- the patterns are already robust.

### Research Integration

Key findings from the research report (01_status-sync-research.md):
- **Sub-issue 1**: Mode A grep patterns in `update-task-status.sh` Phase 3 are robust against wave+tree format. No fix needed.
- **Sub-issue 2**: Mode B regeneration is wired through `skill-implementer` (core) but lean and nvim extension implementers bypass `update-task-status.sh` entirely, using direct jq updates via `postflight-workflow.sh` or inline jq. Mode B never fires for lean4 or neovim tasks.
- **Sub-issue 3**: `link-artifact-todo.sh` itself is safe (anchors on `^### N.`). Contamination comes from agents applying artifact-linking Edit logic to Task Order tree entries because `artifact-linking-todo.md` says "Read the task's entry" without restricting to the `## Tasks` section.
